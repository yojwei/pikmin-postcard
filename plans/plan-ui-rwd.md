# UI 與 RWD 實作計畫（index.html）

## 1) 目標與範圍
- 目標：在不改變核心功能（標題/地點輸入、照片上傳、皮克敏選擇、下載分享）的前提下，提升 `index.html` 的行動版可用性、可讀性與無障礙品質。
- 範圍：僅針對 `index.html` 的版面結構、Tailwind class、ARIA/可及性屬性與互動回饋調整。
- 不含：新增後端 API、重做視覺品牌、引入新框架。

## 2) 現況問題摘要（P1/P2/P3）
- **P1（必修）**
  - 焦點樣式不足，鍵盤操作不清楚。
  - 手機版控制區與明信片區在小螢幕擁擠（版面比例與欄位切分過硬）。
  - 定位按鈕缺少明確「載入中」文字回饋。
- **P2（應修）**
  - 部分元件對比與觸控命中範圍仍可改善。
  - 皮克敏選取狀態辨識度低；小螢幕換行不穩定。
  - 初次使用者 CTA 階層不夠清楚。
- **P3（優化）**
  - 焦點風格與禁用態呈現一致性不足。
  - 極小寬度（<320px）間距與細節可再拋光。

## 3) 分階段實作計畫（Phase 1/2/3）
### Phase 1（先修阻塞：可用性 + 關鍵 RWD）
1. 補齊所有輸入框/按鈕可見 focus 樣式（`focus-visible` ring）。
2. 修正關鍵 ARIA（含圖章價格資訊可被輔助工具讀取）。
3. 定位按鈕加入「定位中…」文字與狀態語意（disabled/aria-busy）。
4. 調整小螢幕版型：控制面板與明信片區在 320/375/480px 可讀可操作。

### Phase 2（體驗強化：資訊層級 + 響應式細修）
1. 優化明信片文字縮放與區塊比例（避免文字擠壓/截斷）。
2. 提升皮克敏選取回饋（不只縮放，加入邊框/底色/對比）。
3. 重整主要 CTA 優先序（先上傳，再調整，再下載/分享）。
4. 檢查並修正按鈕最小可點擊區與對比度。

### Phase 3（品質收斂：一致性與極端裝置）
1. 統一輸入元件 focus 與錯誤/禁用態樣式規則。
2. 微調超小尺寸（<320px）間距、字級與裝飾元素溢位。
3. 完成跨裝置回歸檢查並整理最終樣式基線。

## 4) 驗收標準（含 RWD 與可用性）
- **RWD**
  - 在 320/375/480/768/1024px 皆無水平捲動、無主要內容重疊。
  - 明信片文字可讀（標題/地點不截斷或可預期換行），主要按鈕可完整顯示。
- **可用性**
  - 鍵盤 Tab 可遍歷主要互動元件，且每個元件有清楚 focus。
  - 載入中/禁用態具可視文字或語意提示，避免重複點擊。
  - 首次使用者可在 10 秒內理解流程（設定文字 → 選照片 → 選角色 → 下載）。
- **無障礙**
  - 互動元件具可存取名稱（label/aria-label）。
  - 文字與關鍵 UI 對比達到可讀門檻（至少接近 WCAG AA）。

## 5) 風險與回滾策略
- **風險**
  - 調整版型比例可能影響既有桌機視覺。
  - 響應式 class 改動可能造成局部元件錯位。
  - ARIA 修正若處理不當，可能導致輔助工具朗讀重複。
- **回滾策略**
  1. 以 Phase 為單位提交；每階段可獨立回退。
  2. 先保留舊版 class/結構註記於同次提交，驗證後再清理。
  3. 若發生重大顯示問題，優先回退版型變更，保留無障礙修補（低風險高收益）。
